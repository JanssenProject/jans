# SLSA Artifact Verification

Every release of the Janssen Project ships with [SLSA Build Level 3](https://slsa.dev/spec/v1.0/levels)
provenance attestations alongside the existing cosign signatures. This document
explains how to verify both binary/wheel/WASM artifacts and Docker images.

---

## Prerequisites

### Install `slsa-verifier`

**macOS (Homebrew)**
```bash
brew install slsa-verifier
```

**Go install**
```bash
go install github.com/slsa-framework/slsa-verifier/v2/cli/slsa-verifier@latest
```

**GitHub Release download**  
Download the latest binary from
<https://github.com/slsa-framework/slsa-verifier/releases/latest> and place it
on your `PATH`.

**Verify the installation**
```bash
slsa-verifier version
```

### Install `cosign` (for signature verification)

```bash
brew install cosign         # macOS
# or
go install github.com/sigstore/cosign/v2/cmd/cosign@latest
```

## Verifying Binary / Wheel / WASM Artifacts

Every GitHub Release contains a `.intoto.jsonl` provenance file alongside the
artifact files. The provenance file is named
`<artifact-name>.intoto.jsonl` and is generated by the
`slsa-framework/slsa-github-generator` reusable workflow.

### General command

```bash
slsa-verifier verify-artifact <ARTIFACT_FILE> \
  --provenance-path <ARTIFACT_FILE>.intoto.jsonl \
  --source-uri github.com/JanssenProject/jans \
  --source-tag <RELEASE_TAG>
```

Replace `<RELEASE_TAG>` with the release tag (e.g., `v1.2.0` or `nightly`).

### Examples

**DEB package (Ubuntu 24.04)**
```bash
slsa-verifier verify-artifact \
  jans_1.2.0-stable~ubuntu24.04_amd64.deb \
  --provenance-path jans_1.2.0-stable~ubuntu24.04_amd64.deb.intoto.jsonl \
  --source-uri github.com/JanssenProject/jans \
  --source-tag v1.2.0
```

**RPM package (EL9)**
```bash
slsa-verifier verify-artifact \
  jans-1.2.0-stable.el9.x86_64.rpm \
  --provenance-path jans-1.2.0-stable.el9.x86_64.rpm.intoto.jsonl \
  --source-uri github.com/JanssenProject/jans \
  --source-tag v1.2.0
```

**Python zipapp**
```bash
slsa-verifier verify-artifact \
  jans-linux-ubuntu-X86-64-setup.pyz \
  --provenance-path jans-linux-ubuntu-X86-64-setup.pyz.intoto.jsonl \
  --source-uri github.com/JanssenProject/jans \
  --source-tag v1.2.0
```

**Cedarling WASM tarball**
```bash
slsa-verifier verify-artifact \
  cedarling_wasm_1.2.0_pkg.tar.gz \
  --provenance-path cedarling_wasm_1.2.0_pkg.tar.gz.intoto.jsonl \
  --source-uri github.com/JanssenProject/jans \
  --source-tag v1.2.0
```

**Cedarling Python wheel**
```bash
slsa-verifier verify-artifact \
  cedarling_python-1.2.0-cp311-cp311-manylinux_2_17_x86_64.whl \
  --provenance-path cedarling_python-1.2.0-cp311-cp311-manylinux_2_17_x86_64.whl.intoto.jsonl \
  --source-uri github.com/JanssenProject/jans \
  --source-tag v1.2.0
```

**Cedarling Go shared library**
```bash
slsa-verifier verify-artifact \
  libcedarling_go-1.2.0_x86-64.so \
  --provenance-path libcedarling_go-1.2.0_x86-64.so.intoto.jsonl \
  --source-uri github.com/JanssenProject/jans \
  --source-tag v1.2.0
```

**Cedarling KrakenD plugin**
```bash
slsa-verifier verify-artifact \
  cedarling-krakend-amd64-builder-2.9.0-1.2.0.so \
  --provenance-path cedarling-krakend-amd64-builder-2.9.0-1.2.0.so.intoto.jsonl \
```

**Cedarling UniFFI library / Kotlin binding**
```bash

  --provenance-path libcedarling_uniffi-1.2.0.so.intoto.jsonl \
```

### Also verify the cosign bundle signature

Each artifact also ships with a `.bundle` sidecar (cosign keyless signature).
Verify it alongside the provenance attestation:

```bash
cosign verify-blob \
  --bundle <ARTIFACT_FILE>.bundle \
  --certificate-identity-regexp "https://github.com/JanssenProject/jans" \
  --certificate-oidc-issuer "https://token.actions.githubusercontent.com" \
  <ARTIFACT_FILE>
```

## Verifying Docker Images

Docker images published to GHCR (`ghcr.io/janssenproject/jans/<image>`) carry
SLSA provenance attestations generated by `actions/attest-build-provenance`.
The attestation is stored in the registry and can be verified with
`slsa-verifier` or via `gh attestation verify`.

### With `slsa-verifier`

```bash
slsa-verifier verify-image \
  ghcr.io/janssenproject/jans/<image>@<digest> \
  --source-uri github.com/JanssenProject/jans
```

**Example — auth-server**
```bash
slsa-verifier verify-image \
  ghcr.io/janssenproject/jans/auth-server@sha256:abc123... \
  --source-uri github.com/JanssenProject/jans
```

### With the GitHub CLI (`gh`)

```bash
gh attestation verify \
  oci://ghcr.io/janssenproject/jans/<image>@<digest> \
  --owner JanssenProject
```

### Find the image digest

```bash
docker pull ghcr.io/janssenproject/jans/<image>:<tag>
docker inspect --format='{{index .RepoDigests 0}}' ghcr.io/janssenproject/jans/<image>:<tag>
# or
cosign triangulate ghcr.io/janssenproject/jans/<image>:<tag>
```

### Also verify the cosign signature on Docker images

```bash
cosign verify \
  --certificate-identity-regexp "https://github.com/JanssenProject/jans" \
  --certificate-oidc-issuer "https://token.actions.githubusercontent.com" \
  ghcr.io/janssenproject/jans/<image>@<digest>
```

## Artifact Families and Provenance Files

| Artifact family | Provenance file suffix | Workflow job |
|---|---|---|
| DEB / RPM binary packages | `.intoto.jsonl` | `provenance-binary` |
| Python zipapps (`.pyz`) | `.intoto.jsonl` | `provenance-python` |
| Cedarling WASM tarball | `.intoto.jsonl` | `provenance-wasm` |
| Cedarling Python wheels | `.intoto.jsonl` | `provenance-cedarling-python` |
| Cedarling Go libraries | `.intoto.jsonl` | `provenance-cedarling-go` |
| Cedarling KrakenD plugins | `.intoto.jsonl` | `provenance-cedarling-krakend` |
| Cedarling UniFFI / Kotlin | `.intoto.jsonl` | `provenance-cedarling-uniffi` |
| Demo source zips / Chrome extension | `.intoto.jsonl` | `provenance-demo` |
| Docker images (GHCR) | Registry attestation | `docker` (via `actions/attest-build-provenance`) |
